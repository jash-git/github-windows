合併

您上次的push到執行下一次的push的期間，如果有其他人push更新了遠端數據庫，而您沒有更新您的本地端數據庫，那麼您的push會被拒絕。

這個時候，需要進行合併操作導入其他人的修改歷史，不然push都會被拒絕喔。若沒有合併直接覆蓋歷史記錄的話，其他人push的修改內容（如圖提交C）會消失。

Git執行合併可能會自動合併修改的部分。但是也可能有不能自動合併的時候。在下一頁要講解關於手動合併的方法哦！
